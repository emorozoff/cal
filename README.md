# $horts

Интерактивный калькулятор дохода канала на **YouTube Shorts** от рекламы (AdSense).
Указываешь, сколько видео в месяц выпускаешь, средние просмотры на видео и RPM —
получаешь оценку заработка, с дублем в рублях по актуальному курсу ЦБ РФ.

**Живая версия:** https://emorozoff.github.io/cal/

Один статический `index.html` без зависимостей и сборки — открывается по ссылке с телефона
и с ноутбука.

## Что умеет

- Прямая модель: **видео в месяц × средние просмотры × RPM = доход** (в день / месяц / год, `$` и `₽`).
- Диапазон **пессимист / реалистично / оптимист** (RPM ±30%) — у Shorts он сильно плавает.
- «Доход на 1 млн просмотров» — человеческая единица для копеечного RPM Shorts.
- **Обратный расчёт:** сколько видео в месяц нужно выпускать (при твоих средних просмотрах
  и RPM), чтобы выйти на нужную сумму.
- Курс USD→₽ подтягивается с ЦБ РФ (`cbr-xml-daily.ru`) автоматически при загрузке;
  рядом кнопка «⟳ обновить», чтобы взять свежий курс из интернета. Можно задать курс вручную.
- Тёмная тема по умолчанию + переключатель на светлую (выбор запоминается).
- Расчёт зашивается в URL — можно переслать ссылку с готовыми цифрами.
- Адаптив: на телефоне одна колонка, на ноуте — широкая раскладка в две колонки.

## Как считается

```
Просмотров в месяц = Видео в месяц × Средние просмотры на видео
Доход в месяц ($)  = Просмотров в месяц / 1000 × RPM

Обратный расчёт: Видео в месяц = Цель ($/мес) / (Средние просмотры / 1000 × RPM)
```

RPM для Shorts крошечный (обычно $0.02–0.10 за 1000 просмотров) — монетизация идёт из
общего пула Shorts, и YouTube забирает часть. Поэтому цифры — **оценка по порядку величины**,
а не гарантия дохода.

## Технически

Vanilla HTML/CSS/JS, ноль зависимостей. Темы — через `data-theme` и CSS-переменные;
курс — клиентский `fetch` к ЦБ РФ с ручной правкой и запасным значением.
